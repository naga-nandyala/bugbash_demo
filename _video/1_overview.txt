Change is coming to Azure CLI on macOS. Get ready for two new ways to install — and a game-changing authentication upgrade.

First, Homebrew users will see Azure CLI moving from the old formula install to a Homebrew Cask. 
That means a self-contained, pre-built bundle with native binaries. The new method is designed for reliability and security.

Second, for air-gapped environments and restricted networks, we’re introducing architecture-specific offline tarballs. 
Download from GitHub, extract for your platform — ARM64 or Intel — and point to any compatible Python. 
No package manager needed. It’s flexible, fast, and ready to go where Homebrew can’t.


But that’s not all. 
We're bringing seamless broker based authentication through apple keychain & Company Portal SSO. 
Authenticate with your broker just like you do on Windows, so logging in is smoother and more secure. 
When Company Portal’s installed, Azure CLI uses the broker instead of the browser — it's that easy.


These changes aren’t theoretical. 
We're validating them on real hardware, with live Azure deployments through Ring Zero service.
Now, every install and login works, end-to-end, in the environments you use every day.



Join us in testing. Experience the improvements firsthand. Your feedback will shape the next chapter of Azure CLI on macOS.
